# 流式响应与组件驱动架构设计 (Fluid Responsive & Component-driven Architecture Design)

## 核心哲学 (Core Philosophy)

本项目旨在摆脱传统的“固定像素（Fixed Pixel）”思维，全面转向现代前端的“流式响应（Fluid Responsive）”和“组件复用（Component-driven）”思维。解决目前界面“未铺满屏幕”且“缺乏 React 感”的问题，提升沉浸感、精细度和专业度。

目前的布局依然存在卡牌大小比例失调、元素强制换行、以及大量的无效留白等问题，破坏了沉浸感。要让它具有专业 React 应用的呼吸感和精细度，我们需要从全局到原子组件进行彻底重构。

---

## 架构重构 30 条军规 (30 Rules for Architecture Overhaul)

### 一、 终结“未铺满”：全局流式布局 (Global Layout)

1. **废除外层固定宽度**：检查最外层容器，删除 `max-w-7xl` 或固定的 `width`，改用 `w-screen h-screen overflow-hidden`，让应用真正接管整个浏览器视口。
2. **采用动态列宽分配**：使用 Tailwind 的 Grid（或等效 Vanilla CSS）：`grid-cols-[300px_1fr_250px]`。左右侧边栏固定最小宽度，中间的市场区（`1fr`）会自动吃掉所有剩余的屏幕空间，彻底消除黑边。
3. **释放垂直空间**：中间的 `Card Market` 容器应该使用 `flex-grow` 或 `h-full`，让卡牌区域自然向下延伸，而不是缩在一个暗色方块里。
4. **统一列间距**：左右面板 and 中间市场之间的间隙目前不一致。全局使用统一的 `gap-6` 或 `gap-8` 建立严谨的呼吸空间。

### 二、 核心重灾区：卡牌市场比例 (Card Market Proportions)

5. **强制统一卡牌尺寸**：左侧的牌库（`6 TIER 3`）巨大无比，而右侧翻开的卡片却极其矮小。它们在代码中必须渲染同一个 `<Card />` 组件，保证宽高度绝对 1:1 一致。
6. **引入绝对宽高比 (Aspect Ratio)**：不要给卡牌写死 `height`。给卡牌组件加上 `aspect-[2/3]` 属性。这样当屏幕变宽、卡牌变宽时，高度会自动等比拉伸。
7. **卡牌内部的 Flex 布局重构**：现在的卡牌内容全挤在中间。改为 `flex flex-col justify-between`，把顶部分数和底部成本彻底推向卡牌的上下边缘。
8. **放大关键决策数值**：卡牌左上角的 `4` 和菱形太小了。必须放大至少 1.5 倍，并加粗。这是玩家最关心的信息。
9. **重排底部成本区**：卡牌底部的成本（红3，蓝5）被生硬地截断了。使用 `flex flex-wrap justify-center gap-1 p-2` 建立一个专门的底部栏。
10. **卡牌占位符 (Empty Slots)**：当一行不足 4 张卡时，现在是完全的虚无。应该渲染带有虚线边框的 `<EmptyCardSlot />` 组件，维持 4 列的网格阵型。

### 三、 玩家面板的“数据对齐” (Player Panel Matrix)

11. **修复 Token 与 Bonus 的网格断层**：Gale 面板里，黄球在黄方块上面，但到了蓝球就错位了。必须建立严格的 6 列 Grid (`grid-cols-6`)，确保同一颜色的 Token 和 Bonus 永远在同一垂直线上。
12. **消除无效的背景色块**：Gale 面板里包裹球和方块的那个深灰色长条背景（`bg-gray-800` 等）是纯粹的视觉噪音，删掉它，让球直接浮在面板底色上，会显得高级得多。
13. **缩小 Bonus 方块的体积**：下排代表永久加成的方块太大了。它们的视觉权重不能超过拿在手里的球，将其尺寸缩小到球体的 70%。
14. **对齐 Prestige 徽章**：`Astarion` 名字旁边的威望分数框目前是向下偏移的。使用 `items-center` 让名字和分数框在同一条水平线上完美居中。
15. **统一折叠与展开的宽度**：Astarion 的面板（折叠状态）比 Gale 的面板（展开状态）窄。这在侧边栏设计中是大忌，所有玩家的面板基础宽度必须完全一致。

### 四、 公共资源池的对称性 (Public Pool)

16. **解决第六个球的强制换行**：紫色的 `DQ` 或红色的粉球掉到了第二行，破坏了对称美。缩小球的直径，或者允许父容器使用 `justify-between`，强制 6 个球在一行排开。
17. **强化百搭代币的分割**：黄球和蓝球之间的那条细线太微弱了。加粗分割线，或者给黄球单独加一个底座/光环，明确它的“百搭/黄金”特殊地位。
18. **删除冗余的提示文字**：`Click to take tokens` 这个说明文字打破了右上角的平衡。直接给球加上 `hover:scale-110` 和 `cursor-pointer`，玩家自然知道能点。

### 五、 现代 React 交互反馈 (Interactive UX)

19. **引入悬停状态 (Hover States)**：所有可点击的卡牌必须加上 `transition-transform hover:-translate-y-2 hover:shadow-2xl`，提供物理浮现感。
20. **公共代币的选中状态 (Selection State)**：拿取代币时，玩家需要先选 3 个。选中的代币必须有一个高亮的描边（`ring-4 ring-yellow-400`），再配合一个全局的“确认拿取”按钮。
21. **无效操作的禁用态 (Disabled State)**：当玩家手中的 Token 达到 10 个上限时，公共池的球必须变成黑白或半透明（`opacity-50 grayscale cursor-not-allowed`），直接在视觉上阻断操作。
22. **激活玩家的强视觉锚点**：应该给当前回合玩家的整个面板外框加上缓慢呼吸的发光动画（Pulse Animation）。
23. **添加微交互动画**：使用 Framer Motion（或 CSS 动画），当卡牌被买走时，加一个 0.3 秒的缩放消失动画；新卡牌翻开时加一个 Y 轴翻转动画。

### 六、 贵族区与暗黑奇幻美学 (Patrons & Aesthetics)

24. **重构右侧 Patrons 面板**：去除虚线边框，把它们做成带有华丽金属包边的塔罗牌形状。
25. **提升文字对比度**：如果尚未加载图片，使用高对比度的羊皮纸色（`#E8E2D2`）显示占位符文字。
26. **统一卡牌的背景材质**：使用 CSS 叠加一个极低透明度的噪点背景（Noise Texture）或渐变，让它们有实体纸牌的质感。
27. **修改数字字体 (Typography)**：全站引入一款古典衬线字体（如 `Playfair Display` 或 `Cinzel`）专门用于展示分数和代币数量。
28. **优化霓虹边缘发光**：把发光范围（Blur）调大，颜色调成暗金色或奥术蓝，使其接近魔法光晕而不是霓虹灯管。
29. **统一元素的阴影体系**：背景层最暗，中间的 Market 容器给一点 `shadow-lg`，最上层的卡牌给 `shadow-xl`，用物理投影构建界面的 Z 轴深度。
30. **清理无用的留白 (Dead Padding)**：减少内部元素的垂直 Padding，让信息更加紧凑硬朗。

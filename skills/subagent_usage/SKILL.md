# 🛸 子代理 (Subagent) 使用与任务派发指南

本技能汇总了在当前工作区下管理和派发子代理（Subagent）的规范。它是实现原子化开发的核心超能力。

---

## 🏗️ 一、 顶级 Eager 策略 (Command Execution)

如果你需要子代理在后台静默跑测试、物理执行修复，并且希望它 **零阻断确认、零 UI 弹窗、开启全速自动驾驶**，请设置 `eager` 电量模式！

### 1. 动态 Prompt 抬头法
在调用子代理的 Prompt 最开头添加分隔线口令：
```markdown
---
command_execution_policy: eager
---

你好！请去帮我执行一个任务，并使用 eager 策略全自动跑通弹窗。
```

---

## 🧠 二、 任务派发与“全局视野”注入规范 (Task Prompting)

为了防止子代理“盲人摸象”、只盯着眼前分配的几个文件而写出破坏全局架构的代码，在给子代理派发任务时，**必须在 Prompt 最开头强制注入“全局视野”预习任务**：

```markdown
在动手之前，你必须执行以下预习任务：
1. 阅读 `docs/` 目录下的所有 Markdown 文档，建立全局视野，确保理解项目的设计宪法和模块规范。
2. 扫描 `src/` 目录及其子目录下的 `README.md` 文件，它们会帮助你理解代码结构和发现潜在 Bug。
```

这个模式可以极大地拓宽子代理的上下文，使其在修改局部代码时能够考虑到全局的防守性设计。
